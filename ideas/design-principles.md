# Дизайн-принципы — merge-game

Этот документ содержит требования к дизайну цепочек, которые применяются **до написания промптов**.  
Промпт — финальный шаг. Концепт должен пройти через эти принципы раньше.

---

## 1. Считываемость уровня

**Требование:** игрок должен мгновенно распознать уровень по визуальному облику, без чтения цифры.

**Правила:**

- Каждый уровень обязан иметь **уникальный силуэт**, отличный от соседей L−1 и L+1.
- Размер (`itemSize` 88→188px) — вспомогательный инструмент, не основной. Три одинаковые капли разного размера — не считываются.
- Каждый переход между уровнями должен добавлять **качественный признак**: новый элемент, изменение формы, добавление граней, слияние объектов — не просто "больше".

**Проверка перед промптами:** для каждого уровня явно написано, чем его силуэт отличается от L−1 и L+1. Если написать нечего — дизайн уровня не готов.

---

## 2. Цветовая уникальность между цепочками

Цепочки не должны конфликтовать по доминирующему цвету:

| Цепочка | Цвет |
|---|---|
| Рубин | Красный + золото |
| Кристалл (капли) | Синий (сапфир) |
| Квадраты → Корона | Фиолетовый (аметист) |

Новые цепочки — новый цвет, не пересекающийся с существующими.

---

## 3. Оценка спрайтов — по одному

После генерации каждый спрайт оценивается **отдельно**, по фиксированным критериям:

| Критерий | Вопрос |
|---|---|
| Силуэт | Уникален относительно L−1 и L+1? |
| Считываемость | Понятно что это, без подписи? |
| Цвет | Соответствует цвету цепочки? |
| Качество | Нет артефактов, фон прозрачен, обводка чистая? |

Оценка записывается в секцию `## Спрайты и оценка` файла цепочки — строка на каждый уровень, не итоговая.

**Порядок генерации:** один уровень → оценка → следующий. Не генерировать пачкой. Если уровень не прошёл — переписать промпт и перегенерировать до подтверждения, только потом идти дальше.

**Остановка:** останавливаться только если есть конкретное замечание по спрайту или нужно решение автора. Если спрайт прошёл все критерии — молча идти к следующему уровню.

---

## 4. Дизайн корзинки

Корзинка — не технический параметр, а часть нарратива цепочки. Обсуждается в рамках концепта, **до** `### Проверка считываемости`.

**Что должно быть решено:**

| Вопрос | Пример |
|---|---|
| Откуда берётся L1? | Курятник производит яйца; кристалл капает из пещеры |
| Какой образ корзинки? | Деревянный курятник с курицей; светящаяся пещера |
| Визуально намекает на цепочку? | Корзинка яиц → видна курица; корзинка кристаллов → видны синие кристаллы |
| Промпт для генерации | Записывается в файл цепочки; генерируется отдельным скриптом |

**Требование:** корзинка считается определённой только когда все четыре пункта записаны в `### Корзинка` файла цепочки. Образ в голове — не считается.

---

## 5. Структура файла цепочки

Каждый `ideas/chain-*.md` должен содержать разделы в таком порядке:

1. `## Концепт` — идея, нарратив, логика эволюции
2. `### Корзинка` — откуда берутся предметы, образ, промпт корзинки
3. `### Проверка считываемости` — таблица: уровень / силуэт / чем отличается от соседей
4. `### Уровни` — итоговая таблица с названием и описанием визуала
5. `## Промпты` — только после того, как п.3 заполнен
6. `## Спрайты и оценка` — построчно по каждому уровню после генерации

**Эталонный файл:** [`chain-drops.md`](chain-drops.md) — полный цикл, все разделы заполнены в правильном порядке.

---

## 6. Технические требования к спрайтам

Применяются ко всем спрайтам всех цепочек:

- **Без теней** — ни cast shadow, ни drop shadow. Тени убираются на этапе промпта (`no shadow`, `no drop shadow`) и дополнительно скриптом (Pass 4 в `gen-sprites.js`).
- **Прозрачный фон** — белый фон убирается скриптом (Pass 1–3).
- **Обводка** — чёрный outline 1px, генерируется программно (Pass 5, dilation альфа-маски). В промпте можно просить `thick black outline` для стиля, но нормализуется скриптом.

---

## 7. Exit criteria — переход между стадиями

| Переход | Условие |
|---|---|
| Концепт → Промпты | Корзинка определена (все 4 пункта раздела §4 записаны). Все строки `### Проверка считываемости` заполнены (силуэт + отличие от соседей). |
| Промпты → Генерация | Промпты записаны в файл (не в голове). Минимум один промпт на уровень. |
| Генерация → Готово | Таблица `## Спрайты и оценка` заполнена построчно. Нет строк с `⏳`. |

Пока условие не выполнено — стадия не считается пройденной, даже если работа выглядит готовой.
